Mobile Application

Sleep Tracker App

Role Developer
Duration 25 Hours
Tech Stack Ionic, Angular, IndexedDB
Platform Cross-Platform (iOS Focus)
Project Overview

Getting consistent, quality sleep is a universal challenge. For this project, I developed a comprehensive mobile application designed to track user sleep patterns and daytime energy levels. Built with the Ionic framework, the app implements the Stanford Sleepiness Scale and provides users with actionable data visualizations to help them make informed decisions about their sleeping habits.

View Repository

Target Audience & Design Philosophy

The app was built for anyone looking to monitor and improve their sleep hygiene. Designing a sleep app requires empathy for the user's physical state: they will likely be using the app when they are exhausted, either right before bed or immediately after waking up.

  • Permanent Dark Mode: Implemented a system-wide dark theme to prevent harsh screen glare and eye strain in low-light environments.
  • Frictionless Interactions: Replaced complex forms with a single, massive toggle button for easy operation by a tired user.
  • Platform Conventions: Adhered heavily to iOS guidelines, utilizing familiar mobile paradigms like bottom tab navigation, swipe gestures, and native-feeling modals.

Core Features & Implementation

1. Real-Time & Manual Sleep Logging

To maximize flexibility, the app offers two distinct ways to log overnight sleep:

  • The "Stopwatch" Method: The home screen acts as a useful initial view, featuring a primary "Go to Sleep" / "Wake Up" toggle. This allows users to track their sleep in real time with a single tap.
  • Manual Entry: For users who prefer manual tracking, retroactive logs can be added via the Calendar page.

2. Contextual Sleepiness Tracking

To measure daytime fatigue, the app utilizes the 7-point Stanford Sleepiness Scale.

  • Automated Prompting: When the user taps the "Wake Up" button, a modal automatically triggers with a 1–7 slider. This ensures energy levels are captured at the most critical moment—immediately after waking.
  • Manual Adjustments: Users can also log their sleepiness manually at any point during the day via the main screen or calendar view.

3. Data Visualization & Analytics

Raw data is only useful if it can be easily understood. Users can view their sleep history in three tailored formats:

  • Calendar View: Offers a sorted, date-based look at specific logs.
  • History List: Provides a raw, chronological feed of all logged data.
  • Trends Graph: A visual dashboard that calculates and displays the averages of the last 7 entries, allowing users to quickly spot trends in their sleep duration and energy levels.

Technical Architecture & Data Persistence

A major requirement for this application was ensuring data survived beyond a single session. To achieve this, I integrated Ionic Storage.

  • Persistent Backups: All overnight logs and sleepiness entries are automatically backed up to the device's IndexedDB or Local Storage.
  • Error Prevention & Safety: The app includes strict time-input validation to prevent impossible sleep entries (e.g., waking up before going to sleep). Additionally, "uh-oh" mechanisms like confirmation dialogs are required before a user can delete previous logs, preventing accidental data loss.

Reflection & Takeaways

This project was an excellent exercise in balancing technical execution with empathetic UI/UX design. By focusing on a highly specific user state (exhaustion) and prioritizing intuitive data persistence, the final result is a highly functional, production-ready utility.
